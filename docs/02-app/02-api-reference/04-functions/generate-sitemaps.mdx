You can use the `generateSiteMaps` function to generate multiple sitemaps for your Next.js application.

## Example

For example, to split a sitemap using `generateSitemaps`, return an array of objects with the sitemap `id`. Then, use the `id` to generate the unique sitemaps.

```ts filename="app/product/sitemap.ts" switcher
import { BASE_URL } from '@/app/lib/constants'

export async function generateSitemaps() {
  // Fetch the total number of products and calculate the number of sitemaps
  return [{ id: 0 }, { id: 1 }, { id: 2 }, { id: 3 }]
}

export default async function sitemap({
  id,
}: {
  id: number
}): MetadataRoute.Sitemap {
  // Google limit is 50,000 URLs per sitemap
  const start = id * 50000
  const end = start + 50000
  const products = await getProducts(
    `SELECT id, date FROM products WHERE id BETWEEN ${start} AND ${end}`
  )
  return products.map((product) => ({
    url: `${BASE_URL}/product/${id}`
    lastModified: product.date,
  }))
}
```

```js filename="app/product/sitemap.js" switcher

```

## Returns

The `generateSitemaps` returns an array of objects, each object must have an `id` property.

## URLs

During development, you can view the generated sitemaps on `/route/sitemap.xml/[id]`. For example, `/product/sitemap.xml/1`.

During production, the sitemap is available on `/route/sitemap/[id].xml`.
